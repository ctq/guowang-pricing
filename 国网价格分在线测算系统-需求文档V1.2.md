# 国网价格分在线测算系统需求文档 V1.2

## 0. 文档改进说明

### 0.1 原 V1.0 主要问题

1. 业务目标、商业设想、功能需求混写，开发人员难以判断 MVP 必须交付什么。
2. 多处写“按 Excel 1:1 实现”，但没有定义 Excel 字段映射、计算精度、异常输入、导出格式和验收样例。
3. 算法参数名称不统一，例如 `C/c/a/F1/F2/w1/w2` 在不同算法中含义不同，容易误用。
4. 算法规则汇总表与报价算分模板存在疑似不一致，已确认基准价浮动统一采用 `A × (1+c)`。
5. 数据沉淀边界不清晰。原文只写“自动匿名保存”，但没有说明保存哪些原始数据、是否保存投标人报价、如何脱敏、用户是否可删除。
6. “免费引流”“副业壁垒”等商业表达不适合放在可交付需求主体中，应放到背景或运营规划。
7. 验收标准过粗，缺少可执行的测试口径，例如每种算法至少需要固定样例、边界人数、并列最高/最低、不开标报价等用例。

### 0.2 V1.1 改进原则

1. 以可开发、可测试、可验收为目标。
2. 以 `国网报价算分模板全8套.xlsx` 为计算验收基准。
3. 对 Excel 内部与规则汇总不一致的地方标为“待确认”，不在需求中隐式假设。
4. MVP 先完成在线测算、导入、导出和匿名统计沉淀，暂不做会员、登录、历史查询商业化功能。

### 0.3 V1.2 变更说明

1. **A03（区间复合平均价浮动法）**：确认区间边界**包含临界值**（含本数），从待确认列表移除。
2. **A04（区间复合平均价法（次低价平均））**：确认两点：
   - 区间边界**不包含临界值**（不含本数）。
   - 并列最高价/最低价时，**只去掉其中一个**（按导入顺序取第一个），从待确认列表移除。
3. 更新第 12 节待确认问题列表，移除已决议项。

## 1. 项目概述

### 1.1 产品定位

本系统面向国网物资、服务类投标人员，提供价格评分在线测算能力。用户上传或录入开标报价后，系统根据所选评分方法计算基准价、各投标人价格得分、排名、目标公司得分差，并导出测算结果。

系统同时沉淀匿名化的测算统计数据，用于后续分析不同地区、不同规则、不同限价下的基准价折扣率分布。

### 1.2 目标用户

1. 投标专员：快速核算报价得分、排名和目标公司差距。
2. 投标负责人：评估报价策略，复核 Excel 人工测算结果。
3. 系统运营者：沉淀匿名折扣率数据，后续支持数据分析和增值服务。

### 1.3 MVP 范围

MVP 必须包含：

1. 支持 8 种价格评分方法选择。
2. 支持手工录入报价明细。
3. 支持按模板导入 Excel 报价明细。
4. 支持计算基准价、基准价折扣率、价格得分、排名、目标公司得分差。
5. 支持导出带结果字段的 Excel。
6. 支持匿名沉淀测算摘要数据。
7. 后台保留计算日志，便于核对问题。

MVP 不包含：

1. 用户登录和会员体系。
2. 付费查询历史折扣库。
3. 多人协作、项目管理、审批流。
4. 自动识别任意格式开标记录文件。MVP 只支持标准模板或手工录入。

## 2. 术语定义

| 术语 | 定义 |
| --- | --- |
| 评标价/投标价/报价 | 用于价格分计算的投标人报价。系统内统一字段名为 `bid_price`。 |
| 最高限价/包限价 | 当前包允许的最高报价。系统内统一字段名为 `ceiling_price`。 |
| 初评合格投标人 | 进入价格分计算的投标人。MVP 中由用户通过报价表录入，`不开标` 或空报价不参与计算。 |
| 有效报价 | 按具体算法剔除极值或筛选区间后，用于计算基准价的报价。 |
| 基准价 | 按算法计算得到的价格分比较基准。系统内统一字段名为 `benchmark_price`。 |
| 基准价折扣率 | `benchmark_price / ceiling_price`。若最高限价为空或为 0，则不计算。 |
| 价格分 | 百分制价格得分，负分按 0 处理。 |
| 价格分占比 | 招标文件中价格分权重，用于计算折算后分差。 |
| 目标公司 | 用户重点关注的投标主体，用于展示排名和分差。 |

## 3. 输入需求

### 3.1 项目信息

| 字段 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 招标编号 | 文本 | 是 | 最长 100 字符。 |
| 招标名称 | 文本 | 否 | 最长 200 字符。 |
| 分标编号 | 文本 | 是 | 最长 100 字符。 |
| 包号 | 文本 | 是 | MVP 一次只计算一个包；多包批量计算放到后续版本。 |
| 招标省份 | 下拉 | 是 | 省份列表固定维护，可选“其他”。 |
| 评分方法 | 下拉 | 是 | 从 8 种算法中选择。 |
| 最高限价 | 数值 | 否 | 大于 0，最多 6 位小数。用于折扣率，不参与所有算法的基准价计算。 |
| 价格分占比 | 数值 | 否 | 0 到 1 或 0% 到 100%，用于折算分差。 |
| 目标公司 | 文本 | 否 | 与报价明细中的投标人名称精确匹配；不匹配时提示但不阻断计算。 |

### 3.2 报价明细

报价明细至少包含：

| 字段 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 投标人名称 | 文本 | 是 | 同一包内不建议重复；重复时保留并提示。 |
| 投标价格 | 数值/文本 | 是 | 数值参与计算；值为 `不开标` 时不参与计算，得分显示 0，排名为空。 |

导入文件要求：

1. MVP 支持 `.xlsx`。
2. 默认识别列名：`投标人名称`、`投标人`、`投标价格`、`投标价`、`报价`。
3. 若列名无法识别，前端要求用户手动映射列。
4. 单次计算建议支持 1 到 500 家投标人；低于 1 家不可计算。

### 3.3 通用参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `n1` | 按算法默认 | 报价大于等于基准价时的扣分系数。 |
| `n2` | 按算法默认 | 报价低于基准价时的扣分系数。 |
| `round_scale` | 4 | 价格分默认四舍五入保留 4 位小数。 |

## 4. 评分方法需求

### 4.1 支持算法清单

| 编号 | 算法名称 | 来源工作表 |
| --- | --- | --- |
| A01 | 区间平均价浮动法 | `区间平均价浮动法`、`区间平均价浮动法（折扣率）` |
| A02 | 算术平均值下浮法 | `算术平均值下浮法` |
| A03 | 区间复合平均价浮动法 | `区间复合平均价浮动法` |
| A04 | 区间复合平均价法（次低价平均） | `区间复合平均价法（次低价平均)` |
| A05 | 最低价法 | `最低价法` |
| A06 | 简单算术平均值法 | `简单算术平均值法` |
| A07 | 区间平均下浮双边曲线算法 | `区间平均下浮双边曲线算法` |
| A08 | 区间复合一次平均价法 | `区间复合一次平均价法` |

### 4.2 通用线性扣分公式

适用于 A01、A02、A03、A04、A06、A08，除非算法另有说明。

```text
若 bid_price >= benchmark_price，则 n = n1
若 bid_price < benchmark_price，则 n = n2
score = 100 - 100 × n × abs(bid_price - benchmark_price) / benchmark_price
若 score < 0，则 score = 0
score 四舍五入保留 4 位小数
```

A04 使用扩展公式：

```text
若 bid_price >= benchmark_price，则 m = m_high
若 bid_price < benchmark_price，则 m = m_low
score = 100 - 100 × n × m × abs(bid_price - benchmark_price) / benchmark_price
若 score < 0，则 score = 0
```

### 4.3 A01 区间平均价浮动法

默认参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `w1` | -0.15 | 有效报价下限系数。 |
| `w2` | 0.10 | 有效报价上限系数。 |
| `n1` | 1.0 | 模板默认值；规则汇总中曾出现 1.5，需按招标文件可配置。 |
| `n2` | 0.5 | 模板默认值；规则汇总中曾出现 0.3，需按招标文件可配置。 |
| `c` | 0.01 | 基准价浮动系数，可配置。 |

计算流程：

1. 统计参与计算的投标人数量 `M`。
2. 按 `M` 剔除极值，仅用于计算基准价：
   - `M <= 5`：不剔除。
   - `5 < M <= 10`：剔除 1 个最高价、1 个最低价。
   - `10 < M <= 20`：剔除 2 个最高价、1 个最低价。
   - `20 < M <= 30`：剔除 3 个最高价、2 个最低价。
   - `M > 30`：剔除 4 个最高价、3 个最低价。
3. 计算剔除后报价平均值 `A1`。
4. 计算区间 `[A1 × (1+w1), A1 × (1+w2)]`。
5. 取区间内报价计算平均值 `A2`；若区间内无报价，则回退为剔除极值后的报价集合。
6. 基准价 `benchmark_price = A2 × (1+c)`。
7. 使用通用线性扣分公式计算得分。

待确认：

1. 区间边界是否包含临界值需确认。规则文字写“不含本数”，模板公式表现为边界外/内判断不完全一致。
2. 并列最高价/最低价时剔除数量按 Excel 模板还是按规则文字“只去掉其中一个”执行，需补充样例。

### 4.4 A02 算术平均值下浮法

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `n1` | 1.5 |
| `n2` | 0.3 |
| `c` | 0.01 |

计算流程：

1. 计算所有参与报价的算术平均值 `A1`。
2. 基准价按 Excel 模板实现：`benchmark_price = A1 × (1+c)`。
3. 使用通用线性扣分公式计算得分。

待确认：

1. “下浮法”名称与模板 `1+c` 存在语义差异，应以招标文件或验收模板确认。

### 4.5 A03 区间复合平均价浮动法

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `a` | -0.20 |
| `b` | 0.15 |
| `n1` | 1.0 |
| `n2` | 0.5 |
| `c` | 0 |

计算流程：

1. 若 `M <= 5`，计算所有参与报价平均值 `A1`，基准价为 `A1 × (1+c)`。
2. 若 `M > 5`，计算所有参与报价平均值 `A2`。
3. 计算区间 `[A2 × (1+a), A2 × (1+b)]`，区间边界**包含临界值**（含本数）。
4. 若存在区间内报价，取区间内报价平均值 `A4`；否则取全部报价平均值 `A3`。
5. 基准价为所选平均值 `× (1+c)`。
6. 使用通用线性扣分公式计算得分。

### 4.6 A04 区间复合平均价法（次低价平均）

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `lower_limit` | -0.20 |
| `upper_limit` | 0.10 |
| `m_high` | 1.0 |
| `m_low` | 0.6 |
| `n` | 1.0 |

计算流程：

1. 按 `M` 剔除极值：
   - `M < 10`：不剔除。
   - `10 <= M < 20`：剔除 1 个最高价、1 个最低价。
   - `20 <= M < 30`：剔除 2 个最高价、1 个最低价。
   - `M >= 30`：剔除 3 个最高价、2 个最低价。
2. 并列最高价或最低价时，**只去掉其中一个**（按导入顺序取第一个）。
3. 计算剔除后平均值 `A1`。
4. 计算区间 `(A1 × (1+lower_limit), A1 × (1+upper_limit))`，区间边界**不包含临界值**（不含本数）。
5. 取区间内报价集合 `P`，计算平均值 `A2`。
6. 取 `P` 中最低价 `P_min`。
7. 若 `P` 不为空，基准价 `benchmark_price = (A2 + P_min) / 2`。
8. 若 `P` 为空，基准价回退为全部参与报价平均值。
9. 使用 A04 扩展公式计算得分。

### 4.7 A05 最低价法

计算流程：

1. 基准价为所有参与报价中的最低价。
2. 得分公式：

```text
score = 100 × benchmark_price / bid_price
score 四舍五入保留 4 位小数
```

### 4.8 A06 简单算术平均值法

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `n1` | 1.0 |
| `n2` | 0.8 |

计算流程：

1. 基准价为所有参与报价的算术平均值。
2. 使用通用线性扣分公式计算得分。

### 4.9 A07 区间平均下浮双边曲线算法

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `F1` | 0.8 |
| `F2` | 1.1 |
| `n` | 1.0 |
| `m` | 0.9 |
| `a` | -0.0025 |

计算流程：

1. 计算全部参与报价平均值 `A1`。
2. 计算有效报价区间 `[F1 × A1, F2 × A1]`。
3. 若区间内存在报价，计算区间内报价平均值 `A2`；否则回退为 `A1`。
4. 基准价按 `v1.2` 模板实现：`benchmark_price = A2 × (1-a)`；若区间内无报价，则 `benchmark_price = A1 × (1-a)`。
5. 得分公式：

```text
若 bid_price >= benchmark_price:
score = benchmark_price / bid_price × n × 100

若 bid_price < benchmark_price:
score = bid_price / benchmark_price × m × 100
```

已确认：

1. `国网报价算分模板全8套v1.2.xlsx` 作为当前对账基准，除非模板内部自相矛盾，否则以模板为准。
2. A07 基准价浮动采用模板口径 `1-a`。
3. A07 使用区间内平均值 `A2`；若区间内无报价，则回退为全部参与报价平均值 `A1`。

待确认：

1. 模板中的范围判断列与规则描述存在差异，需补充标准样例。

### 4.10 A08 区间复合一次平均价法

默认参数：

| 参数 | 默认值 |
| --- | --- |
| `w1` | -0.20 |
| `w2` | 0.15 |
| `n1` | 1.0 |
| `n2` | 0.5 |
| `c` | 0 |

计算流程：

1. 计算全部参与报价平均值 `A1`。
2. 计算区间 `[A1 × (1+w1), A1 × (1+w2)]`。
3. 若区间内存在报价，计算区间内报价平均值 `A2`；否则回退为 `A1`。
4. 基准价 `benchmark_price = A2 × (1+c)`；若区间内无报价，则 `benchmark_price = A1 × (1+c)`。
5. 使用通用线性扣分公式计算得分。

## 5. 计算结果展示

### 5.1 页面结果

计算成功后展示：

| 内容 | 说明 |
| --- | --- |
| 基准价 | 保留 6 位小数，可展开查看计算过程。 |
| 基准价折扣率 | `benchmark_price / ceiling_price`，按百分比显示，保留 4 位小数。 |
| 参与计算人数 | 不包含 `不开标` 和空报价。 |
| 有效报价人数 | 用于基准价计算的人数。 |
| 目标公司排名 | 未匹配目标公司时显示“未找到”。 |
| 目标公司得分 | 展示价格分。 |
| 与第一名分差 | `第一名得分 - 目标公司得分`。 |
| 折算后分差 | `分差 × 价格分占比`。 |

### 5.2 明细表

明细表字段：

| 字段 | 说明 |
| --- | --- |
| 排名 | 按价格分降序。 |
| 投标人名称 | 原始输入。 |
| 投标价格 | 原始输入。 |
| 是否参与计算 | `不开标` 或空报价为否。 |
| 是否用于基准价 | 按算法筛选结果。 |
| 价格分 | 计算结果。 |
| 备注 | 剔除原因、不开标、异常提示等。 |

排名规则：

1. 价格分高者排名靠前。
2. 同分时，报价低者靠前。
3. 得分和报价均相同，按导入顺序靠前。

## 6. 导入导出

### 6.1 导入

MVP 支持两种方式：

1. 直接在页面表格录入或粘贴。
2. 上传标准 `.xlsx` 文件并解析报价明细。

导入失败提示必须说明原因，例如：

1. 文件格式不是 `.xlsx`。
2. 未识别投标人列。
3. 未识别报价列。
4. 报价中存在无法解析的非数字文本，且不是 `不开标`。

### 6.2 导出

导出 Excel 至少包含 4 个工作表：

| 工作表 | 内容 |
| --- | --- |
| `测算结果` | 项目信息、基准价、折扣率、目标公司得分、分差。 |
| `报价明细` | 原始报价、参与计算标记、有效报价标记、得分、排名。 |
| `参数设置` | 本次算法参数和值。 |
| `计算过程` | 关键中间值，如 `A1/A2/区间上下限/剔除人数/回退规则`。 |

导出文件名格式：

```text
国网价格分测算_{招标编号}_{分标编号}_{包号}_{YYYYMMDDHHmmss}.xlsx
```

## 7. 数据沉淀

### 7.1 保存范围

MVP 默认只保存匿名摘要，不保存完整投标人报价明细。

保存字段：

| 字段 | 说明 |
| --- | --- |
| `created_at` | 测算时间。 |
| `province` | 招标省份。 |
| `method_code` | 算法编号。 |
| `ceiling_price` | 最高限价。 |
| `benchmark_price` | 基准价。 |
| `discount_rate` | 基准价折扣率。 |
| `bidder_count` | 参与计算人数。 |
| `effective_count` | 用于基准价计算人数。 |
| `param_hash` | 参数摘要，用于排查同一规则下的差异。 |
| `source` | `manual` 或 `xlsx_upload`。 |

不保存字段：

1. 投标人名称。
2. 单家投标报价明细。
3. 招标编号和招标名称原文。

若后续需要保存完整明细，必须在页面明确告知用户，并提供删除机制。

### 7.2 去重规则

同一条沉淀记录满足以下条件时视为重复：

1. `province` 相同。
2. `method_code` 相同。
3. `ceiling_price` 相同，允许误差 `0.000001`。
4. `benchmark_price` 相同，允许误差 `0.000001`。
5. `created_at` 在 10 分钟内。

重复记录不新增，只更新最近使用时间和使用次数。

## 10. 验收标准

### 10.1 功能验收

1. 8 种算法均可选择，切换算法后只展示该算法需要的参数。
2. 手工录入报价可完成测算。
3. 标准 Excel 导入可完成测算。
4. 结果页展示基准价、折扣率、目标公司排名、得分、分差。
5. 导出 Excel 包含测算结果、报价明细、参数设置、计算过程。
6. 匿名摘要数据成功沉淀，且不包含投标人名称和单家报价明细。

### 10.2 计算验收

每种算法至少准备 5 类用例：

1. Excel 模板中的默认样例。
2. 低人数边界样例，例如 `M=5`、`M=10`、`M=20`、`M=30`。
3. 并列最高价/最低价样例。
4. 存在 `不开标` 的样例。
5. 区间内无有效报价、触发回退规则的样例。

验收通过标准：

1. 基准价与 Excel 模板一致，误差不超过 `0.000001`。
2. 价格分与 Excel 模板一致，误差不超过 `0.0001`。
3. 排名与系统定义一致；若 Excel 排名规则不同，需在验收前明确以哪个为准。
4. 所有负分按 0 分处理。

### 10.3 导出验收

1. 导出文件可被 Excel/WPS 正常打开。
2. 导出明细行数与参与输入的报价行数一致。
3. 导出结果中的基准价、得分、排名与页面一致。
4. 导出中包含计算过程关键中间值。

## 12. 待确认问题

开发前必须确认：

1. `区间平均价浮动法（折扣率）` 是否作为独立模式，还是 A01 的报价单位变体。
2. 默认参数以规则汇总表为准，还是以算分模板为准。建议以具体招标文件优先，模板默认值只作为初始值。
3. 是否允许一次提交多个包。建议 MVP 不做，避免输入、导出和沉淀规则复杂化。
4. A01（区间平均价浮动法）区间边界是否包含临界值，以及并列最高/最低剔除口径，需用 Excel 模板样例确认。
